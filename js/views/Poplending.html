Reference Implementation- The Sample Cloud Application

Overview

    Sample cloud app to be used to evaluate different technologies as part of project Tech Stack. For proper evaluation purpose, design of the app should be made close to target architecture as much as possible.

Reference Implementation Objectives

  • Developed following 12 factor microservice design principles.
  • Deployed in JCS.
  • Support dynamic scale out , load balancing ,High Availability.
  • Should serve End to End use case.
  • Use of micro service patterns such as:Circuit Breaker, Cluster Lock, Data Consistency.
  • Should be able to run as Jar/War for OACCS/Bare Metal ( EE/SE).
  • RI should include outcomes of different components of True SaaS project like logging, configuration etc ( EE/SE).
